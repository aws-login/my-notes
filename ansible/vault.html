What is Ansible Vault?

Simple Definition

Ansible Vault is a feature used to encrypt sensitive information in Ansible playbooks.

It protects confidential data such as:

  • Passwords
  • API keys
  • Database credentials
  • SSH keys
  • Tokens

So that other people cannot read them.

Layman Example

Suppose your playbook contains a database password.

Without Vault:

db_user: admin
db_password: mypassword123

Anyone who opens the file can see the password ❌

With Vault:

db_user: admin
db_password: !vault |
          $ANSIBLE_VAULT;1.1;AES256
          326432356334356634...

Password is encrypted and not readable ✅

Why We Use Vault

  • Protect sensitive data
  • Secure automation scripts
  • Safely store secrets in Git repositories
  • Prevent unauthorized access

Example Scenario (Production)

Suppose you automate database deployment. Your playbook contains:

  • DB username
  • DB password
  • API tokens

Instead of writing them in plain text, you store them in a Vault encrypted file.

Creating a Vault File

Create encrypted file:

ansible-vault create secrets.yml

It will ask for a password.

Example content:

db_user: admin
db_password: securepassword

The file is now encrypted.

View Encrypted File

ansible-vault view secrets.yml

Edit Vault File

ansible-vault edit secrets.yml

Encrypt Existing File

ansible-vault encrypt vars.yml

Decrypt File

ansible-vault decrypt vars.yml

Using Vault in Playbook

Example playbook:

- hosts: dbservers
  vars_files:
    - secrets.yml

  tasks:
    - name: Print database user
      debug:
        msg: "{{ db_user }}"

Run playbook:

ansible-playbook deploy.yml --ask-vault-pass

Ansible will ask for the vault password to decrypt the file.

Real Production Example

You automate application deployment on 50 servers.

Sensitive data includes:

  • Database password
  • API token
  • Cloud credentials

All stored in vault encrypted file. This allows you to safely push playbooks to GitHub or GitLab.

Interview Answer (Best Short Version)

Ansible Vault is used to encrypt sensitive data in playbooks. It ensures passwords, tokens, and credentials are stored securely. Vault files can be included in playbooks and decrypted at runtime using a password, making automation safe and production-ready.
