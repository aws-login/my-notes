Setting Up Ansible

Setting up Ansible means installing Ansible on a control machine and configuring it to manage other servers.

Ansible works using SSH, so we only need to install it on one server called the Control Node.

Control Node (Ansible installed)
        |
        | SSH
        |
Managed Nodes (Linux Servers)

Step 1: Install Ansible

On RHEL / CentOS / Rocky / Alma

sudo yum install ansible -y

On Ubuntu

sudo apt update
sudo apt install ansible -y

Check installation:

ansible --version

Example output:

ansible [core 2.x]
python version = 3.x

Step 2: Configure Inventory File

Inventory file contains the list of servers Ansible will manage.

Default location:

/etc/ansible/hosts

Edit the file:

sudo vi /etc/ansible/hosts

Example:

[webservers]
192.168.1.10
192.168.1.11

[dbservers]
192.168.1.20

This means Ansible can manage 3 servers.

Step 3: Setup SSH Passwordless Authentication

Ansible connects to servers using SSH keys.

Generate SSH key:

ssh-keygen

Copy key to server:

ssh-copy-id user@192.168.1.10
ssh-copy-id user@192.168.1.11

Now Ansible can login without password.

Step 4: Test Connectivity

Run:

ansible all -m ping

Example output:

192.168.1.10 | SUCCESS => {
    "ping": "pong"
}

192.168.1.11 | SUCCESS => {
    "ping": "pong"
}

This means Ansible is successfully connected.

Step 5: Run a Simple Command

Example: Check disk usage on all servers.

ansible all -a "df -h"

Example: Install package.

ansible all -m yum -a "name=htop state=present"

Real Production Example

Suppose you manage 100 Linux servers.

Without Ansible:

ssh server1
ssh server2
ssh server3

With Ansible:

ansible all -a "uptime"

All servers execute the command at the same time.
