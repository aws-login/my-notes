What are Loops in Ansible?

Simple Definition

Loops in Ansible are used to repeat a task multiple times using different values.

Instead of writing the same task many times, we use a loop to run it repeatedly.

Think of it like a for loop in programming.

Layman Example

Suppose you want to install 3 packages:

  • vim
  • git
  • htop

Without loop ❌

- name: Install vim
  yum:
    name: vim
    state: present

- name: Install git
  yum:
    name: git
    state: present

- name: Install htop
  yum:
    name: htop
    state: present

Too repetitive.

Using Loop (Better Way)

- name: Install packages
  yum:
    name: "{{ item }}"
    state: present
  loop:
    - vim
    - git
    - htop

What happens:

Install vim
Install git
Install htop

Important Concept: item

item represents each value in the loop list.

vim
git
htop

item will take values one by one.

Example: Creating Multiple Users

- name: Create users
  user:
    name: "{{ item }}"
    state: present
  loop:
    - appuser
    - monitor
    - backup

Result:

appuser created
monitor created
backup created

Real Production Example

- name: Install admin tools
  yum:
    name: "{{ item }}"
    state: present
  loop:
    - vim
    - net-tools
    - htop
    - wget
    - curl

All tools installed automatically.

Loop with Dictionary Example

- name: Create users with shell
  user:
    name: "{{ item.name }}"
    shell: "{{ item.shell }}"
  loop:
    - { name: 'user1', shell: '/bin/bash' }
    - { name: 'user2', shell: '/bin/sh' }

Loop in File Copy Example

- name: Copy config files
  copy:
    src: "{{ item }}"
    dest: /etc/config/
  loop:
    - app.conf
    - db.conf
    - web.conf

1️⃣ Old Loop Syntax: with_items

Earlier versions of Ansible used with_items to repeat tasks.

- name: Install packages
  yum:
    name: "{{ item }}"
    state: present
  with_items:
    - vim
    - git
    - htop

What happens:

Install vim
Install git
Install htop

Here:

item = vim
item = git
item = htop

Example Output

TASK [Install packages]

changed: [server1] => (item=vim)
changed: [server1] => (item=git)
changed: [server1] => (item=htop)

2️⃣ New Loop Syntax: loop

Now Ansible recommends using loop instead of with_items.

- name: Install packages
  yum:
    name: "{{ item }}"
    state: present
  loop:
    - vim
    - git
    - htop

Result is exactly the same.

Comparison

with_items loop
Older Ansible Recommended method
Limited features More flexible
