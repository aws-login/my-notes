Stash Workflow

  1. Check modified files.
  2. Save work using git stash.
  3. Perform urgent task.
  4. Restore changes later.
  5. Delete stash if no longer needed.

Important Stash Commands

git stash

git stash list

git stash apply stash@{0}

git stash pop stash@{0}

git stash drop stash@{0}

git stash clear

Git Blame

Git Blame helps identify who modified specific lines in a file, making it useful for debugging, auditing and troubleshooting. 

app.py

Line 1  ← Developer A
Line 2  ← Developer B
Line 3  ← Developer C
Line 4  ← Developer A

git blame app.py

Use Cases

  • Debugging application issues
  • Tracking code ownership
  • Auditing source code changes
  • Finding when a bug was introduced

Blame Commands

git blame app.py

git blame -L 1,5 app.py

git blame -M app.py

git blame --date=short app.py

Practice: Advanced Git Operations

This guide provides a structured walkthrough of advanced Git operations — Stashing, Tagging, and Blaming — as discussed in the technical training session.

Step-by-Step Action Plan

Module 1: Managing Temporary Work with Git Stash

Use Git Stash to save your working area changes temporarily when switching tasks without committing incomplete code.

  1. Identify Uncommitted Changes: Check modified files with git status.
  2. Save to Temporary Storage: git stash
  3. Verify Stash History: git stash list
  4. Restore Stashed Work:
    • Option A (Keep in Stash): git stash apply <stash_id>
    • Option B (Remove from Stash): git stash pop <stash_id>
  5. Clean Up:
    • Single: git stash drop <stash_id>
    • All: git stash clear

Module 2: Versioning and Labeling with Git Tag

Tags act as labels for specific commits, often used to mark release versions.

  1. Determine Tag Type: Light-weight vs Annotated.
  2. Apply Tag to a Commit:
    • Light-weight: git tag <version> <commit_id>
    • Annotated: git tag -a <version> -m "msg" <commit_id>
  3. Verify Tags: git tag
  4. Sync with Remote:
    • Specific: git push origin <tag>
    • All: git push origin --tags
  5. Remove Tags: git tag -d <tag_id>

Module 3: Auditing Changes with Git Blame

Use Git Blame for debugging and auditing to see which developer modified specific lines.

  1. Investigate File History: git blame <file>
  2. Narrow Down Search: git blame -L <start>,<end> <file>
  3. Analyze on Web Interface: Use GitHub’s Blame button.
