Maven Setup, Installation & Java Project Automation

Apache Maven is a Java-based build automation and project management tool. Maven uses the pom.xml file to manage project configuration, dependencies, plugins, build lifecycle, and artifact creation.

Step 1: Install Java JDK

Maven is a Java-based application and requires the Java Development Kit (JDK) to run.

Java 21 is recommended for modern Java tooling and compatibility with tools such as SonarQube.

Step 4: Configure Maven Environment Variables

Environment variables allow Maven commands to run from any directory. For user-level configuration, Maven variables can be added to the user's .bashrc file. 

vi ~/.bashrc
Back to Top

Add Maven Environment Variables

export PATH=$PATH:/path/to/maven/bin

export M2_HOME=/path/to/maven

export MAVEN_HOME=/path/to/maven

What is Configured in pom.xml?

  • pom.xml

    - It serves as the primary configuration hub where all metadata, dependencies, and build instructions for a Java project are defined.

  • Project name and information

    - It contains the basic identification details of the project, including its display name and general description.

  • Project version

    - It specifies the current release version of the application, which is essential for tracking and managing different project iterations.

  • Group ID

    - The Group ID is a unique identifier for the organization or project group, typically following a reverse domain name convention.

  • Artifact ID

    - The Artifact ID is the specific name of the project or module, used to identify the final output file (like a JAR or WAR).

  • Java dependencies

    - It lists all external Java libraries required by the code, which Maven automatically downloads from the Central Repository to resolve project needs.

  • Database connectors such as MySQL Connector

    - It manages specific third-party integration libraries like the MySQL Connector to ensure the application can connect to databases without manual intervention.

  • Maven plugins

    - It configures reusable software components that extend Maven’s capabilities for specialized tasks like compiling, testing, and code analysis.

  • Build configuration

    - It defines the entire automation lifecycle, providing instructions on how to compile source code, run tests, and package the final executable artifact.

Key Takeaways

  • Maven is a Java build automation and project management tool.
  • Java JDK must be installed before Maven.
  • Downloading Maven binaries gives better version control over the installation.
  • PATH allows the mvn command to run from any directory.
  • M2_HOME and MAVEN_HOME identify the Maven installation directory.
  • pom.xml is the central configuration file of a Maven project.
  • Maven automatically manages project dependencies.
  • The ~/.m2 directory is Maven's default local repository.
  • Missing dependencies are downloaded from configured remote repositories, commonly Maven Central.
  • mvn clean removes previous build output.
  • mvn compile compiles source code.
  • mvn package creates a distributable artifact.
  • mvn install stores the artifact in the local Maven repository.
