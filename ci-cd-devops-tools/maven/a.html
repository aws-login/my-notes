Maven Architecture, Configuration & Project Lifecycle

Maven uses Global Settings, User Settings, a Local Repository, Remote Repositories, pom.xml, and a standard Build Lifecycle to automate Java project creation, dependency management, compilation, testing, and packaging.

Maven Architecture Overview

Maven follows a configuration and repository-based architecture. Maven reads project configuration from pom.xml and combines Global and User settings to determine how the project should be built and where dependencies should be downloaded from.

Global Settings

Global settings are stored inside the Maven installation directory. These settings can affect every Maven user on the system. 

$MAVEN_HOME/conf/settings.xml

User Settings

User settings are stored inside the user's .m2 directory. They are useful for developer-specific configurations such as proxy servers, repository credentials, mirrors, and custom repositories. 

~/.m2/settings.xml

Locate Maven Settings

mvn help:effective-settings | grep -i settings

The effective settings command helps display the final Maven configuration after combining Global and User settings.

Step 4: Create a Maven Project Using Archetype

Maven Archetypes are project templates used to automatically generate a standard Maven project structure. 

Maven Archetype
       |
       |
Project Template
       |
       v
+-----------------------+
| Standard Directories  |
| pom.xml               |
| Java Source Code      |
| Test Source Code      |
+-----------------------+

Generate Maven Project

mvn archetype:generate \
-DgroupId=com.mycompany \
-DartifactId=myproject \
-DarchetypeArtifactId=maven-archetype-quickstart \
-Dversion=1.0 \
-DinteractiveMode=false

Archetype Parameters

``` ```
Parameter Purpose
groupId Organization or company identifier
artifactId Project and artifact name
archetypeArtifactId Defines the Maven project template
version Defines the initial project version
interactiveMode Controls interactive questions during project generation
groupId Example: com.mycompany
artifactId Example: payment-service
Version Example: 1.0

Key Takeaways

  • Maven uses Global and User configuration levels.
  • Global settings apply to every Maven user on the system.
  • User settings apply only to an individual developer.
  • Effective Settings represent the final Maven configuration.
  • The default Maven local repository is ~/.m2/repository.
  • The local repository path can be customized in settings.xml.
  • Maven Archetypes automatically generate standard project structures.
  • groupId normally represents the organization or company namespace.
  • artifactId represents the project or generated artifact name.
  • src/main/java contains application source code.
  • src/test/java contains test source code.
  • mvn compile generates .class files under target/classes.
  • mvn package creates the final project artifact.
  • pom.xml manages project metadata, dependencies, plugins, packaging, and versions.
  • Dependencies can be retrieved automatically from configured remote repositories.
  • JAR is commonly used for Java applications and libraries.
  • WAR is commonly used for Java web applications.
