Key Takeaways

  • Forking is used when contributing to repositories you do not own.
  • Fork creates a GitHub-side copy of a repository.
  • Clone downloads repository to your local machine.
  • Pull Requests enable review before merging changes.
  • Organizations help manage repositories and teams centrally.
  • Teams simplify permission management.
  • Read, Write, and Admin are common repository permissions.
  • PATs are required for authentication in many GitHub workflows.

Phase 1: Contributing to Open Source via Git Forking

Forking is the primary method for contributing to projects you do not own. It creates a server-side copy of a repository in your own GitHub account.

  1. Identify the Project: Locate a public repository you wish to contribute to.
  2. Fork the Repository: On GitHub, click the "Fork" button to create a copy under your account.
  3. Clone to Local Machine: Copy the URL of your forked repository and run git clone.
  4. Modify and Commit: Make changes locally, then use git add and git commit.
  5. Push to Your Fork: Upload changes back to your GitHub fork with git push.
  6. Submit a Pull Request (PR): Open a PR from your fork to the original repository for review.

Phase 2: Setting Up GitHub Organizations and Teams

Organizations provide isolated zones to manage multiple projects and developers efficiently.

  1. Create an Organization: Click "+" on GitHub → "New Organization".
  2. Create a Team: Navigate to the "Teams" tab and create a new team.
  3. Invite Members: Add members via usernames or email addresses.
  4. Create/Add Repositories: Create repos within the organization or assign existing ones to teams.
  5. Manage Permissions:
    • Read: View and clone only.
    • Write: Push changes.
    • Admin: Full control.

Phase 3: Collaborative Workflow within an Organization

  1. Accept Invitation: Members must accept the email invite.
  2. Generate Personal Access Token (PAT): Required for private repos via CLI.
  3. Clone and Push: Clone repos with credentials. Push allowed only if "Write" access is granted.

git pushUploads local commits to the remote repository on GitHub.git push origin master
