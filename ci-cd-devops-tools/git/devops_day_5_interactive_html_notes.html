Git Branching, Remote Collaboration & Rebase

Branching allows developers to work independently on features, bug fixes, and enhancements without affecting production code. Rebase helps maintain a clean commit history by replaying commits on top of the latest branch changes.

Phase 1: Local Branching Workflow

  1. Initialize repository.
  2. Create baseline commits on main/master.
  3. Create feature branch.
  4. Switch to feature branch.
  5. Develop and commit changes.
  6. Test feature.
  7. Merge back into main branch.

Why Rebase?

  • Maintains clean project history.
  • Reduces unnecessary merge commits.
  • Makes Git logs easier to understand.
  • Supports fast-forward merges.

Rebase Workflow

  1. Update local main branch.
  2. Switch to feature branch.
  3. Execute rebase.
  4. Resolve conflicts if any.
  5. Merge feature branch.
git switch main

git pull origin main

git switch feature

git rebase main

Key Takeaways

  • Never develop directly on the production branch.
  • Use feature branches for enhancements and bug fixes.
  • Pull Requests enable code review and approvals.
  • Rebase creates a cleaner history than repeated merges.
  • Rebase helps enable fast-forward merges.
  • Feature branches support parallel development.
  • GitHub collaboration relies heavily on branching workflows.

Phase 1: Local Branching and Merging

Use this workflow to develop a feature locally and integrate it into your main codebase.

  1. Initialize a Repository: git init
  2. Establish Baseline Code: Create files and commit them with git add . and git commit -m "[message]".
  3. Create a Feature Branch: git branch feature-login
  4. Switch to the New Branch: git checkout feature-login
  5. Develop and Commit: Modify files and commit regularly.
  6. Merge Back to Master:
    • Switch back: git checkout master
    • Merge: git merge feature-login

Phase 2: Remote Branch Management (GitHub)

Use this workflow when collaborating on remote repositories like GitHub.

  1. Clone the Repository: git clone https://github.com/user/repo.git
  2. Branch and Work Locally: Create a branch and make changes.
  3. Push the Specific Branch: git push origin feature-login
  4. Open a Pull Request (PR): Use GitHub UI to open a PR.
  5. Review and Merge: Admin reviews and merges changes into main.

Phase 3: Advanced Versioning with Git Rebase

Use Rebase to maintain a clean, linear project history.

  1. Identify Base Divergence: Another developer updates master.
  2. Execute Rebase: On your branch, run git rebase master.
  3. Linearize History: Rebasing rewrites history for a clean timeline.
