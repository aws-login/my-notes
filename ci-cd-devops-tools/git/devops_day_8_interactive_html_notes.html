Summary

Git branches allow developers to work independently. Changes are combined using merge operations. Depending on commit history, Git performs either Fast Forward Merge or Three-Way Merge. If the same code is modified in multiple branches, merge conflicts occur and require manual resolution.

Fast Forward Merge

Three-Way Merge

Occurs when both branches contain independent commits. Git creates a merge commit. 

          C4 ---- C5
         /          \
C1 ---- C2 ---- C3   M1
         \
          C6 ---- C7


M1 = Merge Commit
  
git switch master

git merge feature

Merge Conflict

Conflicts occur when the same line is modified differently in multiple branches. 

Main Branch

Database = MySQL


Feature Branch

Conflict Resolution Steps

  1. Open conflicted file.
  2. Review both versions.
  3. Choose required code.
  4. Remove conflict markers.
  5. Save file.
  6. Run git add .
  7. Create merge commit.
git add .

git commit -m "resolved conflict"

GitHub Conflict Resolution

  1. Push feature branch.
  2. Create Pull Request.
  3. GitHub detects conflict.
  4. Click Resolve Conflicts.
  5. Edit conflicting code.
  6. Mark as Resolved.
  7. Commit Merge.
  8. Merge Pull Request.
git push origin feature

git push origin main

Key Takeaways

  • Fast Forward Merge moves the branch pointer forward.
  • Three-Way Merge creates a merge commit.
  • Conflicts require manual resolution.
  • GitHub provides browser-based conflict resolution.
  • Branching enables parallel development.
  • Merge strategies are essential Git concepts.

Practice:

Phase 1: Executing a Fast Forward Merge

A fast-forward merge occurs when the parent branch has no new commits since the child branch was created, allowing Git to simply move the pointer forward.

  1. Initialize Repository: git init
  2. Create Initial Commits: Add a file, then git add and git commit -m "message".
  3. Create a Feature Branch: git checkout -b feature
  4. Develop in Feature Branch: Modify code and commit changes.
  5. Merge into Parent: git switch master then git merge feature.

Phase 2: Resolving Local Merge Conflicts (Three-Way Merge)

  1. Prepare Conflict Scenario: Modify the same line in both master and feature branches.
  2. Attempt the Merge: git merge feature → conflict occurs.
  3. Identify Conflict Markers: 
    <<<<<<< HEAD
(master branch content)
=======
(feature branch content)
>>>>>>> feature
  4. Manual Resolution: Edit file, remove markers, keep desired code.
  5. Finalize Resolution: git add . → git commit -m "resolved conflict".

Phase 3: Resolving Conflicts on GitHub

  1. Push Branches: git push origin main, git push origin feature.
  2. Open Pull Request: Attempt merge on GitHub.
  3. Identify Conflict: GitHub shows “has conflicts that must be resolved.”
  4. Web-Based Resolution: Use Resolve conflicts editor, fix code.
  5. Commit and Merge: Mark resolved → Commit merge → Merge PR.
