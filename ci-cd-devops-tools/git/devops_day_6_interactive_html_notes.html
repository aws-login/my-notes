Git Squash, Fetch, Pull & Revert

These advanced Git operations help maintain clean commit history, synchronize remote changes safely, and undo mistakes without deleting project history.

Git Squash Merge

Squash Merge combines multiple commits from a feature branch into a single commit before merging into the main branch. 

Feature Branch

C1 → C2 → C3 → C4


git merge --squash feature


Main Branch

C1 → C5

(C5 contains all changes)
Back to Top

Squash Workflow

  1. Switch to main branch.
  2. Execute squash merge.
  3. Review staged files.
  4. Create final commit.
git switch main

git merge --squash feature

git status

git commit -m "Feature Completed"

Git Revert

Git Revert safely undoes changes by creating a new commit that reverses a previous commit.

Revert preserves history. Reset rewrites history. 
Original History

C1 → C2 → C3


git revert C3


New History

C1 → C2 → C3 → R1

R1 = Revert Commit
Back to Top

git revert HEAD

Revert Specific Commit

Resolution Steps

  1. Open conflicted file.
  2. Review both versions.
  3. Remove conflict markers.
  4. Save file.
  5. Stage resolved file.
  6. Continue revert process.
git add .

git revert --continue

Key Takeaways

  • Squash merges create cleaner commit history.
  • Fetch is safer because it does not automatically merge.
  • Pull = Fetch + Merge.
  • Revert preserves project history.
  • Revert is preferred over Reset on shared branches.
  • Conflict markers must be manually resolved.
  • git show helps inspect commit details before reverting.
