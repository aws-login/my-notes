Creating a Repository in GitHub

  1. Login to GitHub
  2. Click New Repository
  3. Enter repository name
  4. Select Public or Private
  5. Click Create Repository

Remote Repository Commands

Command Purpose
git remote Display configured remotes
git remote -v Display remote URLs
git remote add origin URL Add remote repository
git remote remove origin Remove remote repository
git remote rename old new Rename remote

Clone Repository

git clone https://github.com/user/project.git

Downloads the complete repository including source code, commit history, branches, and remote configuration.

SSH Authentication

Generate SSH Key

ssh-keygen -t rsa -C "email@example.com"

Display Public Key

cat ~/.ssh/id_rsa.pub

Test Connection

ssh -T git@github.com

Git Push

First Push

git push -u origin main

Uploads code and creates upstream tracking.

Normal Push

git push

Git Pull

git pull

Downloads and merges latest changes from GitHub.

Git Reset

Reset Type Command Result
Soft Reset git reset --soft <commit-id> Commit removed, changes remain staged
Mixed Reset git reset <commit-id> Commit removed, changes moved to working area
Hard Reset git reset --hard <commit-id> Commit removed, changes permanently deleted

Key Takeaways

  • GitHub hosts remote repositories.
  • SSH provides secure authentication.
  • git clone downloads repositories.
  • git push uploads commits.
  • git pull retrieves updates.
  • git reset helps undo commits.
  • Understanding remote repositories is essential for DevOps workflows.
