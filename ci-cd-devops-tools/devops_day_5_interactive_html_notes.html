Git Branching, Remote Collaboration & Rebase

Branching allows developers to work independently on features, bug fixes, and enhancements without affecting production code. Rebase helps maintain a clean commit history by replaying commits on top of the latest branch changes.

Git Branching Overview

A branch is an independent line of development. Developers create branches to isolate work and avoid impacting the stable production branch (main/master).

Phase 1: Local Branching Workflow

  1. Initialize repository.
  2. Create baseline commits on main/master.
  3. Create feature branch.
  4. Switch to feature branch.
  5. Develop and commit changes.
  6. Test feature.
  7. Merge back into main branch.
git branch feature-login

git switch feature-login

git add .

git commit -m "Login Feature"

git switch main

git merge feature-login

Remote Collaboration Steps

  1. Clone repository.
  2. Create feature branch.
  3. Develop feature.
  4. Commit changes.
  5. Push feature branch.
  6. Create Pull Request.
  7. Review and approve.
  8. Merge into main branch.
git clone REPOSITORY_URL

git checkout -b feature-payment

git add .

git commit -m "Payment Module"

git push origin feature-payment

Git Rebase

Rebase replays commits from one branch on top of another branch. It creates a cleaner, linear commit history. 

Before Rebase

Main

C1 ----- C2 ----- C3 ----- C4

           \
            \
             C5 ----- C6
               Feature


git rebase main


After Rebase

C1 ----- C2 ----- C3 ----- C4 ----- C5 ----- C6
Back to Top

Why Rebase?

  • Maintains clean project history.
  • Reduces unnecessary merge commits.
  • Makes Git logs easier to understand.
  • Supports fast-forward merges.
Back to Top

Rebase Workflow

  1. Update local main branch.
  2. Switch to feature branch.
  3. Execute rebase.
  4. Resolve conflicts if any.
  5. Merge feature branch.
git switch main

git pull origin main

git switch feature

git rebase main

Key Takeaways

  • Never develop directly on the production branch.
  • Use feature branches for enhancements and bug fixes.
  • Pull Requests enable code review and approvals.
  • Rebase creates a cleaner history than repeated merges.
  • Rebase helps enable fast-forward merges.
  • Feature branches support parallel development.
  • GitHub collaboration relies heavily on branching workflows.
