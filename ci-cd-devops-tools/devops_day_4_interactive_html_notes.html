Git Reset

Git Reset allows developers to move changes backward through Git's three areas:

  • Commit Area
  • Staging Area
  • Working Area
Commit Area
     |
     |
git reset
     |
     v
Staging Area
     |
     |
git reset
     |
     v
Working Area

Types of Reset

``` ```
Reset Type Behavior
Soft Reset Moves commit back but keeps files staged
Mixed Reset Moves files back to working area (default)
Hard Reset Permanently deletes changes

Soft Reset

git reset --soft COMMIT_ID
Changes remain in the Staging Area.

Mixed Reset

git reset --mixed COMMIT_ID
Changes move back to the Working Area.

Hard Reset

git reset --hard COMMIT_ID
Warning: Permanently deletes changes from local system.

Key Takeaways

  • Origin is the default alias for a remote repository.
  • git remote -v displays fetch and push URLs.
  • Cloning automatically configures remote connections.
  • git push uploads local commits to GitHub.
  • Soft Reset preserves staged files.
  • Mixed Reset moves files back to the working area.
  • Hard Reset permanently removes changes.
  • you can set the git URL using command "git remote add origin URL" , let say you ran "git remote -v" and its showing "https://" git url but you want to use SSH way , you can update the SSH URL using "git remote add origin URL" command.
