Probe

Liveness Probe

Liveness Probe tells the Kubernetes cluster to restart the container.

Suppose there is a deadlock happening inside a Kubernetes cluster and the Docker container does not start.

Readiness Probe

Readiness Probe works as a doorkeeper for incoming traffic.

It is responsible for determining if the pod is ready to receive traffic. If the pod is not ready, the Readiness Probe informs the Kubernetes cluster to remove the pod. If the pod is healthy and ready to receive traffic, it informs the cluster to add it to the load balancer.

Startup Probe

Startup Probe is responsible for checking the running application inside the container.

It verifies if the application is running inside the container and informs the Kubernetes cluster of its status. If the application is not running, the Startup Probe communicates with the Liveness and Readiness probes to prevent them from starting.
