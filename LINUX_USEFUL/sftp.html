SFTP SETUP

Note : We are automating a monthly file transfer from SUSE-LINUX to ALMA-LINUX using cron and password less rsync over SSH.

Source where report.txt generating - SUSE-LINUX ; IP - 192.168.1.10

Destination where report.txt pushed for application use - ALMA-LINUX ; IP - 192.168.1.9

Phase 1: Configure Key-Based Authentication (No-Password Login) on ALMA-LINUX ; IP - 192.168.1.9

To allow the automated cron job to connect securely without a human typing a password, you must share SSH keys between the machines.

  • Log into destination as your standard user.
  • Generate a secure deployment key pair:
ssh-keygen -t ed25519 -N "" -f ~/.ssh/id_ed25519
  • Append the public key directly into the SFTP server's authorized file:
cat ~/.ssh/id_ed25519.pub | ssh user@192.168.1.10 \
'mkdir -p /app/sftp/data/.ssh && \
cat >> /app/sftp/data/.ssh/authorized_keys && \
chmod 700 /app/sftp/data/.ssh && \
chmod 600 /app/sftp/data/.ssh/authorized_keys && \
chown -R user:users /app/sftp/data/.ssh'

Phase 2: Create the Automated Script on linux-6yta

  • Create a dedicated directory:
mkdir -p /home/user/scripts
  • Create and open the production script file:
vi /home/user/scripts/sftp_pull.sh
  • Paste the following code:
#!/bin/bash
# Production SFTP Download Automation

REMOTE_SERVER="192.168.1.10"
REMOTE_USER="user"
REMOTE_FILE="/uploads/report.txt"

######################################################################################################
# The remote path must be "/uploads/report.txt" because the SFTP server is using a Chroot Jail.      #
# A Chroot Jail renames /app/sftp/data/ to / for that session.                                       #
######################################################################################################

LOCAL_DEST="/home/user/"

# Execute secure batch download
sftp -b <(echo "get ${REMOTE_FILE} ${LOCAL_DEST}") ${REMOTE_USER}@${REMOTE_SERVER}
  • Apply strict permissions:
chmod 700 /home/user/scripts/sftp_pull.sh

Phase 3: Test the Configuration Manually on ALMA-LINUX ; IP - 192.168.1.9

  • Trigger the script manually:
/home/user/scripts/sftp_pull.sh
  • Verify the file:
ls -l /app/sftp/data/report.txt

Phase 4: Configure the Production Cron Schedule

  • Open cron configuration:
crontab -e
  • Add execution string:
0 * * * * /home/user/scripts/sftp_pull.sh > /dev/null 2>&1

Test Script to run on SUSE-LINUX ; IP - 192.168.1.10

This script creates report.txt, rotates old files, and asks for new content.

#!/bin/bash

BASE="report.txt"

# Find highest index among report.txt-N files
max=0
for f in ${BASE}-*; do
    if [[ -f "$f" ]]; then
        idx=${f##*-}
        if [[ "$idx" =~ ^[0-9]+$ ]]; then
            (( idx > max )) && max=$idx
        fi
    fi
done

# Rotate in reverse order
for ((i=max; i>=1; i--)); do
    old="${BASE}-${i}"
    new="${BASE}-$((i+1))"
    if [[ -f "$old" ]]; then
        mv "$old" "$new"
    fi
done

# Rotate base file
if [[ -f "$BASE" ]]; then
    mv "$BASE" "${BASE}-1"
fi

# Ask user for content
echo -n "Enter content for the new report: "
read content

# Create new report.txt
echo "$content" > "$BASE"

# Set ownership to user:users
chown user:users "$BASE"

echo "Rotation complete. New report.txt created with ownership user:users."
