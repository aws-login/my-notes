Adding Secondary IP on Cluster Nodes (RHEL)

Environment: Red Hat Enterprise Linux cluster nodes.

Primary Host IP (Permanent)

Each node has its own primary IP configured in the interface file:

Node1

/etc/sysconfig/network-scripts/ifcfg-eth0
IPADDR=10.203.201.73
PREFIX=24

Node2

/etc/sysconfig/network-scripts/ifcfg-eth0
IPADDR=10.203.201.74
PREFIX=24

These IPs are:

  • Primary host IP
  • Permanent
  • Loaded automatically after reboot

Adding Secondary IP Online (Recommended Method)

Use NetworkManager command:

nmcli con modify "System eth0" +ipv4.addresses 10.203.193.79/24
nmcli device reapply eth0

Advantages:

  • No downtime
  • Applied immediately
  • Persistent after reboot
  • Recommended for production

Temporary Method (Not Persistent)

ip addr add 10.203.193.79/24 dev eth0

Characteristics:

  • Works immediately
  • Lost after reboot
  • Used only for testing

Old Method (Network Scripts)

Edit configuration file:

vi /etc/sysconfig/network-scripts/ifcfg-eth0

Example:

IPADDR1=10.203.193.79
PREFIX1=24

Then restart network.

This method works but is not preferred when NetworkManager is running.

Verification Commands

Check IP addresses:

ip a show eth0

Check NetworkManager configuration:

nmcli con show "System eth0" | grep ipv4.addresses

Check routing source IP:

ip route get 8.8.8.8

Adding Secondary IP on Debian (Classic Networking)

This works on systems using the classic Debian networking (/etc/network/interfaces).

Add Secondary IP Online (Immediate)

First add the IP live:

ip addr add 192.168.1.20/24 dev eth0

Verify:

ip a show eth0

You will see:

192.168.1.10  → primary
192.168.1.20  → secondary
  • ✅ IP becomes active immediately
  • ✅ No downtime
  • ❌ Not persistent yet

Make It Permanent

Edit:

vi /etc/network/interfaces

Example:

auto eth0
iface eth0 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    gateway 192.168.1.1

# secondary IP
up ip addr add 192.168.1.20/24 dev eth0
down ip addr del 192.168.1.20/24 dev eth0

This means:

  • When interface starts → IP added
  • When interface stops → IP removed

No Restart Needed

Because you already added the IP using ip addr add, no restart is required.

After reboot, Debian will automatically add it.

Verify Configuration

Check running IPs:

ip a

Check configuration:

cat /etc/network/interfaces

Why This Method is Preferred

Method Downtime Persistent
ip addr add ❌ No ❌ No
interfaces file only ⚠ restart needed ✅ Yes
ip + interfaces method ✅ No downtime ✅ Yes

This is the standard production method used by many Linux admins.

Example Final Output

eth0
 ├── 192.168.1.10  (primary)
 └── 192.168.1.20  (secondary)

💡 Senior Linux Admin Tip

To quickly see all secondary IPs on a server:

ip -brief addr

Example output:

eth0  UP  192.168.1.10/24 192.168.1.20/24

Adding Secondary IP on SUSE (No Downtime)

This applies to systems running SUSE Linux Enterprise Server.

Add Secondary IP Online (No Downtime)

Run:

ip addr add 192.168.1.20/24 dev eth0

Example:

ip addr add 10.203.193.79/24 dev eth0

Verify:

ip a show eth0

You will see:

192.168.1.10   → primary
192.168.1.20   → secondary
  • ✅ Immediate
  • ✅ No network restart
  • ❌ Not persistent yet

Make It Permanent (SUSE Method)

Edit the interface configuration file:

/etc/sysconfig/network/ifcfg-eth0

Add the secondary IP using IPADDR_1 format.

Example:

IPADDR='192.168.1.10/24'
IPADDR_1='192.168.1.20/24'

If you already have multiple IPs it may look like:

IPADDR='192.168.1.10/24'
IPADDR_1='192.168.1.20/24'
IPADDR_2='192.168.1.30/24'

Apply Without Restarting Network

Instead of restarting networking, run:

wicked ifreload eth0

SUSE uses wicked as the network manager.

This reloads the configuration without dropping the interface.

Verify

ip a show eth0

or

wicked show all

Summary (Production Safe Method)

Step Command
Add IP online ip addr add IP/24 dev eth0
Make permanent edit /etc/sysconfig/network/ifcfg-eth0
Apply safely wicked ifreload eth0

Result

  • Secondary IP added online
  • No downtime
  • IP remains after reboot
